EventService
service in module w20CoreEnv

Description

The EventService provides functions to emit and listen to application-wide events. Behind the scenes the EventService, uses AngularJS events on the root scope.

Methods

  • broadcast(eventType, args)

    Broadcast an application-wide event.

    Parameters
    ParamTypeDetails
    eventTypeString

    The type of the event.

    argsString

    The arguments of the event (which will be passed to any listener function).

  • emit(eventType, args)

    Emit an application-wide event.

    Parameters
    ParamTypeDetails
    eventTypeString

    The type of the event.

    argsString

    The arguments of the event (which will be passed to any listener function).

  • on(eventType, listener, listenerScope)

    Register a listener for a given type of event. Listeners have a scope :

    • Application scope (by default or 'application' as the listenerScope argument). The listener will live for the duration of the application. No automatic deregistration will be done.
    • View scope ('view' as listenerScope argument). The listener will live for the duration of the current view. On view change, it will be automatically deregistered.
    Parameters
    ParamTypeDetails
    eventTypeString

    The type of the event.

    listenerString

    A function that will be called upon event reception.

    listenerScopeString

    The scope of the listener.

    Returns
    Function

    The listener unregistration function.
